Multi-tenant

Chaque syndic client = 1 enregistrement Organization. Toutes les tables ont un champ organization_id qui isole les données. Un utilisateur ne peut jamais accéder aux données d'une autre organisation.

1. Authentification & Comptes

Inscription

Route POST /register — Crée simultanément :

  • 1 Organisation (avec slug unique auto-généré)
  • 1 abonnement trial de 3 mois (90 jours)
  • 1 compte admin avec le mot de passe saisi

Règles mot de passe : min 8 caractères + 1 majuscule + 1 chiffre.

Connexion

Rate limiting : 5 tentatives par minute. Session permanente avec timeout 30 min. Vérification organisation active au login.

3 rôles

RôleAccès
superadminGestion de toutes les organisations SyndicPro
adminGestion complète de sa résidence
residentEspace personnel : paiements, tickets, profil

2. Gestion de la copropriété

Appartements

CRUD complet. Chaque appartement appartient à un Bloc (ex : Bloc A). Champs : numéro, redevance mensuelle, crédit automatique.

Système de crédit automatique

Comment ça fonctionne

Si le montant payé ne couvre pas exactement un nombre de mois entiers, le surplus est conservé en crédit sur l'appartement et déduit automatiquement du prochain paiement.

Exemple : Redevance 100 DT/mois — Paiement 350 DT → 3 mois payés + 50 DT crédit.
Paiement suivant 80 DT → Total disponible 130 DT → 1 mois payé + 30 DT crédit.

7. Alertes impayés

Détection automatique des résidents avec retard. Marquage "envoyé" pour suivre les relances. Les 5 dernières alertes sont visibles sur le tableau de bord admin.

8. Paiement en ligne Konnect

Flux résident

  1. Résident voit le bouton "Payer en ligne" (si mois impayés + Konnect configuré)
  2. Clic → POST /konnect/pay → appel API Konnect → redirection vers page paiement
  3. Après paiement → Konnect redirige vers /konnect/success?payment_ref=xxx
  4. Vérification API Konnect → enregistrement automatique dans Payment si confirmé
  5. Notification WhatsApp admin + résident

Flux admin

Génération de liens partageables depuis la page Encaissements. Historique complet sur /konnect/links.

Statut lienSignification
pendingLien généré, paiement non encore effectué
completedPaiement confirmé par Konnect, enregistré en base
failedPaiement échoué ou annulé

10. Assistant Claude AI

Configuration requise

Variable d'environnement ANTHROPIC_API_KEY à ajouter sur Render.
Compte séparé de Claude Pro — créer sur console.anthropic.com.

Modèle utilisé : claude-haiku-4-5 (rapide, économique ~$0.001/conversation).

Contexte injecté en temps réel à chaque requête :

  • Nombre d'appartements payés/impayés du mois
  • Liste détaillée des impayés avec résidents
  • Solde trésorerie (historique complet)
  • Nombre de tickets ouverts

Historique de conversation : 10 derniers échanges conservés côté navigateur.

14. Espace résident

Accessible via /residents pour les utilisateurs avec rôle resident.

  • Statut clair : à jour ✅ ou nombre de mois impayés avec montant dû
  • Bouton "Payer en ligne" (si Konnect configuré et mois impayés)
  • Informations appartement : numéro, redevance, prochain mois dû, crédit disponible
  • Historique des 6 derniers mois (payé / impayé)
  • Accès aux tickets
  • Changement de mot de passe
